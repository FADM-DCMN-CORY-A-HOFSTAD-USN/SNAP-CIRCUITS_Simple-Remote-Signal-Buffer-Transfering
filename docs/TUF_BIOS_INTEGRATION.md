# ASUS TUF BIOS / UEFI Integration Standard
**Target:** Boeing Berry-Compliant TUF Assembly Line
**Objective:** Configure native motherboard fTPM to secure the Snap Circuit.

To avoid manufacturing discrete TPM chips, the Double Latch Gate Buffer relies entirely on the ASUS TUF native firmware TPM. Plant engineers must ensure every motherboard leaves the line with the following strict UEFI parameters.

## Step 1: Enable Firmware TPM (fTPM / PTT)
Do not install external TPM headers. We are using the CPU's secure enclave.
1. Boot the TUF terminal and press `F2` or `Del` to enter the BIOS.
2. Press `F7` to enter **Advanced Mode**.
3. Navigate to **Advanced > AMD fTPM Configuration** (or **PCH-FW Configuration** for Intel).
4. Set TPM Device Selection to **Firmware TPM** (or enable **PTT**).

## Step 2: Enforce Strict Secure Boot
The `hex_tuf_uefi_bridge.py` script will hard-fault if Secure Boot is bypassed.
1. Navigate to the **Boot** menu.
2. Select **Secure Boot**.
3. Set OS Type to **Windows UEFI mode** (Enforces strict WHQL signature checks, which we will override with our own keys).
4. Change Secure Boot Mode from *Standard* to **Custom**.

## Step 3: Injecting the Revolutionary Technology Hex Keys
To ensure the motherboard only boots when the Snap Circuit is physically intact and verified by our Python scripts:
1. Enter **Key Management** under the Secure Boot Custom menu.
2. Clear the default factory Secure Boot keys.
3. Append the `.db` and `.dbx` hexadecimal signatures generated by the Master Univac Orchestrator.
4. Press `F10` to Save and Reset.

The motherboard is now Berry Compliant, Mil-Spec, and natively acts as the cryptographic vault for the mechanical qubit.
